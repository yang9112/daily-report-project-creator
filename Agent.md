# Agent.md — 仓库文档与数据上传规范

## 项目特定规范

本项目是日报项目创建工具的技能包，遵循以下规范：

### ✅ 允许上传
- **源代码文件**: *.js, *.json 技能实现和配置
- **文档文件**: README.md, Agent.md, SKILL.md
- **脚本模板**: scripts/ 目录下的创建脚本
- **参考文档**: references/ 目录下的参考文档
- **模板文件**: Dockerfile, 配置模板
- **CI/CD配置**: .github/workflows/ 目录

### ❌ 严格禁止上传
- **API密钥和配置**: .env, config.json (含真实密钥)
- **生成的项目**: output/ 目录下的用户项目
- **日志文件**: *.log, logs/ 目录下的运行日志
- **临时文件**: .tmp, cache/ 目录下的缓存文件
- **个人配置**: .local, .config 目录下的个人设置
- **敏感信息**: 任何含token、password、secret的文件

### 🔄 自动化规则

#### 配置文件管理
- 统一使用 `.example.json` 作为模板
- 真实配置文件必须在 `.gitignore` 中排除
- 环境变量通过 `.env.example` 模板提供

#### 输出文件处理
- 生成的用户项目文件不提交到仓库
- 脚本输出和日志文件本地处理
- 缓存和临时数据及时清理

#### 脚本执行规范
- 脚本可读取配置但不写入配置
- 生成的项目在外部目录创建
- 避免在技能目录内产生副作用

## Agent行为约束

### 限制条件
1. **不修改用户系统**: 不在系统目录写入文件
2. **不污染环境**: 清理临时文件和进程
3. **不泄露隐私**: 不读写用户私人文件
4. **不滥用权限**: 仅访问必要的系统资源

### 安全措施
1. **输入验证**: 验证项目名称和配置参数
2. **路径安全**: 防止路径遍历攻击
3. **权限检查**: 确认操作权限后再执行
4. **回滚机制**: 失败时自动清理

## Git配置

### 分支策略
- `main`: 主分支，稳定版本
- `develop`: 开发分支，功能开发
- `feature/*`: 功能分支，特定功能

### 提交信息规范
```
type(scope): description

feat(creator): 添加批量创建功能
fix(config): 修复配置文件生成问题
docs(readme): 更新使用说明
refactor(scripts): 重构项目创建逻辑
```

### 发布流程
1. 功能开发在 `feature/*` 分支
2. 合并到 `develop` 分支测试
3. 完成后合并到 `main` 分支
4. 创建发布标签
